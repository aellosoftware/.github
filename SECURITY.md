# Security policy

Aello Software takes responsible vulnerability reports seriously.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Email security@aellosoftware.com with:

- The affected product, repository, version, or commit
- A clear description of the issue and potential impact
- Reproduction steps or a minimal proof of concept
- Relevant logs or screenshots with secrets and personal data removed
- Any suggested mitigation, if known

Use a subject such as `Security report: project name - short summary`. Do not send passwords, access tokens, private keys, or production customer data.

## What to expect

Aello will review reports as capacity allows, may request additional information, and will coordinate remediation and disclosure when the report is actionable. No fixed response or resolution time is promised unless a specific product publishes one.

Please allow a reasonable opportunity to investigate and address the issue before public disclosure.

## Supported versions

Each repository should identify its supported versions. When no version policy is published, assume only the latest maintained release is eligible for security fixes.
