# Governance

Aello Software projects are independently maintained by Cameron Low. Cameron is the final human owner for product direction, repository policy, production changes, and public releases.

## Where work happens

- Public issues and pull requests on GitHub are welcome for discussion and contribution.
- Gitea is the canonical source forge. An accepted GitHub contribution is imported to a Gitea branch, reviewed and merged there, and only the approved sanitized tree is mirrored back to GitHub.
- Linear is the canonical system for internal projects, release gates, blockers, evidence, and approvals.
- The private Aello Catalog maps products, repositories, deployments, coordination, support routes, and release policy.

Public contributors do not need access to the internal systems. Maintainers will preserve attribution when importing accepted contributions.

## Decisions and review

Routine fixes follow the repository contribution policy. Material product, security, compatibility, licensing, governance, or operational changes require explicit maintainer review. Decisions may consider user impact, maintenance burden, security, compatibility, evidence quality, and long-term stewardship.

A review, passing check, or agent recommendation is not release approval. Cameron must explicitly approve every production or public release.

## Security and conduct

Use the private route in SECURITY.md for vulnerabilities and SUPPORT.md for non-public support. Conduct reports follow CODE_OF_CONDUCT.md. Do not place credentials, customer data, private vulnerability details, private URLs, or environment values in public issues or pull requests.

## Changes to this policy

Governance changes may be proposed through an issue or pull request in the relevant repository. Adoption requires Cameron's explicit approval.
