# Public repository license policy

Public visibility does not by itself grant permission to copy, modify, or redistribute software.

An Aello repository is open source only when it contains an explicit LICENSE file. The license is selected per project during release-readiness review based on the software, dependencies, business model, contribution model, and distribution goals.

Before public release, every repository must:

- Include an approved LICENSE file or be clearly marked as source-available/proprietary.
- Include required third-party notices and preserve upstream license terms.
- Confirm that dependencies, bundled assets, fonts, datasets, and generated material may be redistributed.
- Identify the copyright holder consistently.
- Document any separate terms for hosted services, trademarks, documentation, or commercial use.

Do not copy a license from another Aello project without confirming it fits the new project. License changes after public release require explicit review because contributors and downstream users may rely on the existing terms.

Questions about licensing should be sent to legal@aellosoftware.com.
