# Contributing to Aello projects

Thank you for helping improve an Aello Software project.

## Before you begin

Read the repository README, setup instructions, license, security policy, and any project-specific contribution notes. Repository-specific guidance overrides this default.

For a bug, search existing issues before opening a new one. For a feature or substantial change, start with an issue so the problem, scope, and approach can be discussed before implementation work begins.

## Pull requests

Keep pull requests focused and explain both the problem and the chosen solution. Include relevant tests, documentation, migration notes, screenshots, or reproducible validation steps. Avoid unrelated formatting or dependency changes.

Before requesting review:

- Run the repository's documented checks.
- Confirm no secrets, credentials, private URLs, or personal data are included.
- Update user-facing documentation when behavior changes.
- Record breaking changes and operational impact clearly.
- Complete the pull-request template.

By contributing, you agree that your contribution may be distributed under the license declared by that repository.

## Conduct and security

Follow the Aello community code of conduct. Do not report security vulnerabilities in public issues; follow the repository's security policy or email security@aellosoftware.com.
